セキュリティモデル

このエディタは完全ローカル実行を前提にしています。CDN、外部配信JavaScript、外部配信CSS、ネットワークAPIは使いません。

  • Content Security Policyで通信、埋め込み、フォーム送信、外部スクリプトを禁止。
  • Markdown内のHTMLは実行せず、文字としてエスケープ。
  • JavaScript URL、危険なプロトコル、外部画像読み込みをブロック。
  • 画像はPNG/JPEG/GIF/WebPのみ。挿入画像はassetsフォルダへの相対参照にし、File System Access APIで許可されたフォルダ内の相対パスだけを表示。
  • 下書き、設定、リンク許可ドメインはブラウザのlocalStorageに保存。リンク許可ドメインはユーザーが選んだJSON設定ファイルから読み込み、JSONとして書き出せます。許可した設定フォルダは起動時に自動読み込みし、同じJSONへ上書き保存できます。File System Access APIのフォルダハンドルはIndexedDBに保存。
  • Markdown保存は、フォルダ許可済みなら開いているMarkdownファイルへ上書きし、未許可時はユーザー操作によるダウンロード保存。
  • 貼り付け/ドロップ画像は、フォルダ許可済みの場合だけMarkdownファイル横のassetsフォルダへ保存。外部アップロードや同期は行いません。

ローカルデータ管理

この操作はブラウザ内の下書き・設定を削除します。保存済みMarkdownファイルや assets フォルダ内の画像は削除しません。

外部リンクの許可ドメイン

1行に1ドメインを入力します。許可されていない外部リンクはプレビューでリンク化しません。

設定ファイルを読み込むと、この一覧を更新してlocalStorageへ保存します。

Markdownファイルを選択

選択したフォルダ内のMarkdownファイルを開きます。

リンクを挿入

許可ドメイン制は維持されます。許可されていない外部リンクや危険なURLは挿入しません。
